Настройку делаем на HQ-SRV

Скачиваем пакет bind9

`apt install bind9 -y`

# Основная конфигурация

`nano /etc/bind/named.conf.options`

![{C951F0A8-3701-4A41-BD84-A3B0F3B73684}](https://github.com/user-attachments/assets/702a9b00-fb8a-447c-8dde-f9228b0a2968)

# Обозначение зон

`nano /etc/bind/named.conf.default-zones`

Удаляем или комментируем имеющиеся зоны и добавляем свои.

![image](https://github.com/user-attachments/assets/5e0c1025-61d9-4130-8035-7603a0da7fa7)

# Настройка зон

Копируем шаблоны для зон из существующего файла

`cp /etc/bind/db.empty /etc/bind/au-team.db`

`cp /etc/bind/db.empty /etc/bind/au-team_rev.db`

`cp /etc/bind/db.empty /etc/bind/au-team_rev2.db`

Перейдём к редактированию конфигов

`nano /etc/bind/au-team.db`

![image](https://github.com/user-attachments/assets/f2df1704-04c9-4318-88d5-cdd03fdc8b73)

`nano /etc/bind/au-team_rev.db`

![image](https://github.com/user-attachments/assets/04177f8b-ebb7-49ce-b7f2-f59b8d2d7290)

`nano /etc/bind/au-team_rev2.db`

![image](https://github.com/user-attachments/assets/f123c940-7d05-47a3-b426-d3737ecceba4)

# Проверка конфигураций

`named-checkconf -z`

![image](https://github.com/user-attachments/assets/76b246d5-6f32-436c-a9bc-88e435d1df94)

`systemctl restart bind9`

# Настройка на BR-SRV

Стираем всё и прописываем ip-адрес DNS сервера, в нашем случае HQ-SRV

`nano /etc/resolv.conf`

![{EC84ADCC-8C17-463D-934A-566B585B4CAC}](https://github.com/user-attachments/assets/dd8eaaa5-6d8c-4694-8948-e01a64c2b794)

`systemctl restart networking`

# Проверка

Проверку DNS производим также на BR-SRV, прямую зону путём ввода команды `host` и имени устройства с доменом, обратную проверяем с вводом ip-адреса устройства.
